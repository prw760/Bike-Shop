Theory

If you're running a website, it's always good to have a separate page for each object in the database, but obviously, it's a bit too much to create the pages manually! With Django, it can be reasonably simple. Suppose you have a website for a library with the main page with a list of books. You want a separate page for each book to provide details: author, genre, short description, etc. Let's see how to do this step by step:

  • In views.py, you could use the following code:

class BookView(View):
    def get(self, request, pk, *args, **kwargs):
        pk = self.kwargs['pk']
        book = Book.objects.filter(id=pk).first()
        return render(request, 'bookdetails.html', {'book':book})

In this code snippet, we've created the class-based view BookView and defined the get method that processes get requests. From self.kwargs['pk'] you can read the primary key (pk). Next, pk can filter books to find the book object we are looking for.

  • With the following code piece, you can add the urlpatterns in urls.py:

path('books/<int:pk>/', BookView.as_view(), name='book_detail')

We've added a new path to handle the URLs in the format books/1/, books/2/, etc.

  • To list links of every title on the main page, add one code line:

{% for book in books %}
    <p><a href="{% url 'book_detail' book.pk %}">{{book.title}}</a></p>
{% endfor %}

The application uses the link defined in the book_detail path with the corresponding pk number. For example, the book with pk=1 forms the link books/1/.

There's also an alternative approach that uses generic display views: ListView and DetailView. You can read about them in the official documentation.

Description

To make our website more informative for customers, add a separate page for each bike to display details of a particular model. The main page should include links; users should be able to click on a bike's link that leads to the relevant page with a description.

Objectives

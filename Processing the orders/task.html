Description

In this stage, your task is to ensure that the customers' orders are processed, the submitted information is saved, and the bike shop staff can see the orders in the admin panel. It would also be great to inform your customers that you have their order and provide them with the order number.

In the previous stage, you've created a form for ordering a bike; now, your application needs to accept and process the submitted data via POST requests. Let's break it down into steps; the following things should happen once a customer clicks on the order button:

  • Create an order object with information on the customer — name, surname, and phone number. Orders should be visible in the admin panel;
  • The number of bike parts — update frames, seats, tires, and basket. Update only the number of the right bike parts — if your shop uses black, blue, and white frames, but the ordered bike comes with a black frame, decrease the black frame by one. Also, don't forget that each cycle has two tires;
  • Redirect the customer to a new page — localhost:8000/order/order_no/, where order_no is the automatically generated order id.

The localhost:8000/order/order_no/ page is a simple page with two elements:

  • An <h2> tag with text The Thank you for your order! heading under the <h2> tag;
  • Your order number is *order_no*. We will call you once your bike is ready! under the <p> tag.

That's it! You can go ahead and test it. Use the python manage.py runserver command to start the application. In particular, don't forget to test if the number of elements gets updated and orders appear in the admin panel.

Objectives
  • Ensure that the order information is saved through POST requests following the Description section;
  • Once the order is made, redirect the customer to localhost:8000/order/order_no/, a new page with two elements: the thank-you message and the information on the order.
